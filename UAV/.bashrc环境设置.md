# 设置PX4在ros、gazebo下的仿真环境变量
在~/.bashrc文件末尾添加以下内容
```bash
source ~/catkin_ws/devel/setup.bash
source ~/PX4_Firmware/Tools/setup_gazebo.bash ~/PX4_Firmware/ ~/PX4_Firmware/build/px4_sitl_default
export ROS_PACKAGE_PATH=$ROS_PACKAGE_PATH:~/PX4_Firmware
export ROS_PACKAGE_PATH=$ROS_PACKAGE_PATH:~/PX4_Firmware/Tools/sitl_gazebo
```
1. source ~/catkin_ws/devel/setup.bash
    加载 ROS 工作空间的配置脚本
    设置 ROS 环境变量和命令自动补全功能
    使系统能够识别您自己构建的 ROS 软件包
2. source ~/PX4_Firmware/Tools/setup_gazebo.bash ~/PX4_Firmware/ ~/PX4_Firmware/build/px4_sitl_default
    设置 Gazebo 仿真环境（PX4 使用的仿真器）
    添加 PX4 的无人机模型和仿真插件到 Gazebo 的搜索路径
    指定 PX4 固件路径和编译输出目录
3. export ROS_PACKAGE_PATH=$ROS_PACKAGE_PATH:~/PX4_Firmware
    将 PX4 固件目录添加到 ROS 包搜索路径
    使 ROS 能够找到 PX4 相关的功能包和节点

4. export ROS_PACKAGE_PATH=$ROS_PACKAGE_PATH:~/PX4_Firmware/Tools/sitl_gazebo
    将 PX4 的 Gazebo 仿真插件目录添加到 ROS 包搜索路径
    使 ROS 能够识别和使用 PX4 的 Gazebo 仿真环境

sudo sh -c 'echo "deb http://packages.osrfoundation.org/gazebo/ubuntu-stable `lsb_release -cs` main" > /etc/apt/sources.list.d/gazebo-stable.list'
